# 规格说明书 SPEC.md

## 1. 项目概述

| 项目 | 内容 |
|------|------|
| 项目名称 | Web Agent Chat System |
| 项目类型 | 全栈 Web 应用 (Vue + Express) |
| 核心功能 | 基于 Web 的聊天窗口 Agent 系统，支持对接 OpenAI 兼容的大模型 |
| 目标用户 | 开发者/测试人员/需要快速部署 AI Agent 的用户 |

---

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 聊天功能
- [x] 多轮对话：支持上下文连续对话
- [x] 创建新会话：支持多个独立会话
- [x] 会话列表：展示所有历史会话
- [x] 删除会话：支持删除不需要的会话

#### 2.1.2 大模型对接
- [x] OpenAI 兼容接口：支持任意 OpenAI 兼容格式的 LLM
- [x] 自定义 Endpoint：用户可配置 API 地址
- [x] API Key 管理：支持在设置中配置
- [x] 流式响应：支持 Server-Sent Events (SSE) 流式输出

#### 2.1.3 Agent 配置
- [x] System Prompt：支持设置 Agent 角色和行为指令
- [x] 默认配置：使用模型默认参数（不提供参数调整 UI）

#### 2.1.4 MCP Server 支持
- [x] MCP Server 配置管理：支持添加、编辑、删除 MCP Server 连接
- [x] MCP 协议客户端：实现 MCP JSON-RPC 协议，支持与 MCP Server 通信
- [x] 工具调用：支持在聊天中使用 MCP Server 提供的工具
- [x] 并发工具调用：支持同时调用多个工具
- [x] 工具调用显示：在聊天界面显示工具调用状态和结果

#### 2.1.5 数据持久化
- [x] SQLite 存储：会话消息持久化到 SQLite
- [x] 会话导出：支持导出单个会话为 JSON
- [x] 会话导入：支持导入 JSON 文件恢复会话

### 2.2 前端功能

#### 2.2.1 聊天界面
- [x] 消息展示：区分用户消息和 AI 回复
- [x] 代码高亮：使用 highlight.js 或 prism 进行代码渲染
- [x] 复制功能：一键复制消息内容
- [x] 文件上传：支持上传文件作为上下文（基础版本：作为文本发送）
- [x] 主题切换：支持亮色/暗色主题
- [x] 打字机效果：流式输出时逐字显示

#### 2.2.2 设置页面
- [x] LLM 配置：设置 API Endpoint、API Key、模型名称
- [x] Agent 配置：设置 System Prompt
- [x] 主题切换：亮色/暗色主题
- [x] MCP Server 管理：添加、编辑、删除、测试 MCP Server 连接

## 3. API 设计

### 3.1 REST API

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/sessions` | 获取所有会话列表 |
| POST | `/api/sessions` | 创建新会话 |
| GET | `/api/sessions/:id` | 获取会话详情（含消息） |
| DELETE | `/api/sessions/:id` | 删除会话 |
| POST | `/api/chat` | 发送消息（支持流式） |
| GET | `/api/export/:id` | 导出会话 |
| POST | `/api/import` | 导入会话 |
| PUT | `/api/settings` | 保存设置 |
| GET | `/api/settings` | 获取设置 |
| GET | `/api/settings/mcp-servers` | 获取 MCP Server 列表 |
| PUT | `/api/settings/mcp-servers` | 批量更新 MCP Servers |
| POST | `/api/settings/mcp-servers` | 添加 MCP Server |
| PUT | `/api/settings/mcp-servers/:id` | 更新单个 MCP Server |
| DELETE | `/api/settings/mcp-servers/:id` | 删除 MCP Server |
| POST | `/api/settings/mcp-servers/test` | 测试 MCP Server 连接 |

### 3.2 GraphQL Schema

```graphql
type Session {
  id: ID!
  title: String!
  createdAt: String!
  messages: [Message!]!
}

type Message {
  id: ID!
  role: String!
  content: String!
  createdAt: String!
}

type Settings {
  apiEndpoint: String
  apiKey: String
  model: String
  systemPrompt: String
  theme: String
}

type Query {
  sessions: [Session!]!
  session(id: ID!): Session
  settings: Settings
}

type Mutation {
  createSession(title: String!): Session!
  deleteSession(id: ID!): Boolean!
  sendMessage(sessionId: String!, content: String!): String!
  updateSettings(input: SettingsInput!): Settings!
  importSession(json: String!): Session!
}

input SettingsInput {
  apiEndpoint: String
  apiKey: String
  model: String
  systemPrompt: String
  theme: String
}
```

## 4. 数据模型

### 4.1 Database Schema (SQLite)

```sql
-- 会话表
CREATE TABLE sessions (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);

-- 消息表
CREATE TABLE messages (
  id TEXT PRIMARY KEY,
  session_id TEXT NOT NULL,
  role TEXT NOT NULL,  -- 'user' | 'assistant'
  content TEXT NOT NULL,
  created_at TEXT NOT NULL,
  FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE
);

-- 设置表
CREATE TABLE settings (
  id INTEGER PRIMARY KEY CHECK (id = 1),
  api_endpoint TEXT DEFAULT '',
  api_key TEXT DEFAULT '',
  model TEXT DEFAULT 'gpt-3.5-turbo',
  system_prompt TEXT DEFAULT '',
  theme TEXT DEFAULT 'light',
  mcp_servers TEXT DEFAULT '[]'
);
```

## 5. 验收标准

### 5.1 功能验收

| 场景 | 验收条件 |
|------|----------|
| 创建会话 | 点击"新建会话"后，创建一个空会话并切换到该会话 |
| 发送消息 | 发送消息后，消息显示在聊天区域，AI 回复显示在下方 |
| 多轮对话 | 发送多条消息后，每条消息都保留在聊天记录中 |
| 流式响应 | AI 回复时逐字显示，模拟打字机效果 |
| 会话导出 | 导出的 JSON 包含完整会话信息，可正确导入 |
| 会话导入 | 导入 JSON 后，会话和消息正确恢复 |
| 主题切换 | 切换主题后，界面配色正确切换并持久化 |
| 代码高亮 | AI 回复中的代码块正确高亮显示 |
| 复制功能 | 点击复制按钮后，内容复制到剪贴板 |
| 文件上传 | 上传文件后，文件名显示在输入框附近 |
| MCP Server 连接 | 添加 MCP Server 后，可以测试连接是否成功 |
| MCP 工具调用 | 当 LLM 请求工具时，系统执行工具并显示结果 |
| 工具调用错误 | 工具调用失败时，显示错误信息 |

|------|------|

| 测试覆盖 | 核心 API 和业务逻辑必须有测试覆盖 |
