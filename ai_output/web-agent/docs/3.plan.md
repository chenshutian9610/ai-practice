# 技术方案 PLAN.md

## 1. 技术选型

### 1.1 技术栈总览

| 层级 | 选型 | 替代方案考虑 |
|------|------|-------------|
| 前端框架 | Vue 3 (Composition API) | - |
| 前端状态管理 | Pinia | Vuex (Pinia 更轻量) |
| UI 组件库 | 原生实现 (MVP 轻量) | Element Plus / Naive UI |
| 后端框架 | Express + TypeScript | Fastify (Express 更成熟) |
| 数据库 | better-sqlite3 (同步 API) | Prisma (更重) |
| REST 框架 | 原生 Express | tsoa / express-openapi |
| GraphQL | Apollo Server Express | graphql-yoga (更轻) |
| 代码高亮 | highlight.js | Prism (功能类似) |

### 1.2 设计权衡

#### 数据库层：better-sqlite3 vs Prisma
- **选择**：better-sqlite3
- **理由**：SQLite 是文件数据库，性能足够；better-sqlite3 同步 API 简单，不需要额外学习 ORM 语法；MVP 阶段不需要复杂查询

#### GraphQL：Apollo vs 纯 REST
- **选择**：Apollo Server Express
- **理由**：支持自动生成 schema，与 REST 共存；社区成熟，文档完善

#### 前端状态管理：Pinia vs Vuex
- **选择**：Pinia
- **理由**：Vue 官方推荐，更轻量，TypeScript 支持更好

## 3. 核心模块设计

### 3.1 LLM 调用服务 (services/llm.ts)

```typescript
interface LLMConfig {
  endpoint: string;
  apiKey: string;
  model: string;
}

interface ChatMessage {
  role: 'system' | 'user' | 'assistant';
  content: string;
}

// 流式响应使用可读流
function chat(config: LLMConfig, messages: ChatMessage[]): ReadableStream {
  // 调用 OpenAI 兼容接口
  // 返回 SSE 流
}
```

**设计理由**：
- 抽象 LLM 调用逻辑，便于切换不同 provider
- 流式响应使用 ReadableStream，保持 Node.js 和浏览器端一致

### 3.2 数据库服务 (services/database.ts)

```typescript
// 使用 better-sqlite3 同步 API
const db = new Database('./data/web-agent.db');

// 初始化表
```

**设计理由**：

## 4. API 实现细节

### 4.1 REST API 路由

```
/api/sessions
  GET    -> 返回所有会话列表
  POST   -> 创建新会话 { title }

/api/sessions/:id
  GET    -> 返回会话详情 + 消息
  DELETE -> 删除会话

/api/chat
  POST   -> 发送消息 { sessionId, content }
         -> 返回: 同步模式返回完整回复
         -> 流式: 使用 text/event-stream

/api/export/:id
  GET    -> 返回会话 JSON 文件下载

/api/import
  POST   -> 导入会话 JSON { json }
         -> 返回新会话

/api/settings
  GET    -> 获取设置
  PUT    -> 更新设置
```

### 4.2 流式响应实现

```typescript
// server/src/routes/chat.ts
app.post('/api/chat', async (req, res) => {
  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');

  const stream = await llm.chatStream(config, messages);

  for await (const chunk of stream) {
    res.write(`data: ${JSON.stringify({ content: chunk })}\n\n`);
  }
  res.write('data: [DONE]\n\n');
  res.end();
});
```

## 7. 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| LLM API 不稳定 | 添加超时 + 重试机制，记录错误日志 |
| 流式响应中断 | 前端处理连接断开，显示错误提示 |
| SQLite 并发 | MVP 阶段单用户，暂不考虑并发 |
| 主题切换闪烁 | 初始化时读取主题，避免 FOUC |
